## [1.0.0-alpha.5] — Pre-release

### Crop Images in Results

Campaign results now include crop images and thumbnails. Users can view thumbnails in the results table, expand rows to see full crop images, and highlight entries on the source page. Progressive reveal and lightbox features enhance the review experience.

- Added crop image endpoint and thumbnail_url in results.
- Added thumbnail column to results table and CampaignResultResponse interface.
- Added expandable rows and crop lightbox to results table.
- Added per-entry highlight and progressive reveal on source page.
- Clipped thumbnail/crop to entry row and source page lightbox.

### Sorting and Pagination

Results tables now support client-side sorting and efficient SQL-level pagination. This improves navigation and performance for large result sets.

- Added client-side sort for results table.
- Added SQL-level pagination and GROUP BY for results.

### Prediction Display

Predictions are now adaptively truncated based on score, making tables more readable and focused.

- Added adaptive score-based prediction truncation.
- Added renderPredictionsTable and renderExpandedCropImage pure functions.

### Security and Validation

Security and validation improvements address HTML sanitization and storage path checks. Focus traps and fixes respond to audit findings.

- Sanitized HTML output and added focus trap.
- Fixed validation of resolved path against storage base.
- Addressed security findings in results and frontend.

### Fixed

- Centralized API URL and fixed relative URL 404s.
- Added crop metadata to campaign results endpoint.
- Fixed crop_router test import shadowing.
- Updated featureFlags test environment to match base-url convention.
